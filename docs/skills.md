# Скіли

Конектор дає асистенту **доступ** до даних. Скіл дає йому **метод** — сталий спосіб виконати задачу: за яким чеклістом писати, як форматувати звіт, яких помилок уникати. Один раз підключили — і асистент застосовує цей метод у кожній відповіді, без нагадувань у промпті.

Для керівника найкорисніші скіли — ті, що стосуються тексту й документів, які потім читає хтось інший: листи, нотатки в CRM, підсумки зустрічей, пости.

## Олюднення тексту (humanizer)

Асистент за замовчуванням пише «як асистент»: штампи, надлишкові узагальнення, шаблонні звороти на кшталт «у світі, що постійно змінюється». Скіл **humanizer** прибирає ці ознаки й наближає текст до природного людського письма.

Правило просте: застосовуйте його щоразу, коли текст піде далі за ваш чат — у лист, у документ, у CRM, у публікацію.

**Як підключити (Claude):**

```
/plugin marketplace add blader/humanizer
/plugin install humanizer@humanizer
```

Далі викликайте його командою `/humanizer` або згадкою в запиті. Для інших асистентів є крос-платформна установка:

```
npx skills add blader/humanizer
```

Офіційне джерело: [github.com/blader/humanizer](https://github.com/blader/humanizer).

**Як застосувати без установки.** Якщо встановити скіл не можна, додайте його логіку прямо в промпт:

> Після того, як напишеш текст, застосуй принципи humanizer: прибери типові ознаки AI-письма — штампи, надлишкові узагальнення, шаблонні звороти й «воду». Залиш просте, конкретне формулювання.

## Чи вказувати скіли в промптах

Так, якщо скіл не встановлено як окремий інструмент. Тоді ви або згадуєте його в промпті (`застосуй humanizer`), або вписуєте його правила текстом, як вище. Якщо скіл встановлено в асистенті — він застосовується автоматично, окремо в промпті вказувати не треба.

## Свій метод як скіл

Будь-яку повторювану інструкцію можна оформити як скіл і не повторювати щоразу:

- **Бренд-голос** — єдиний тон і словник у всіх листах та постах.
- **Структура звітів** — один формат вихідних документів: розділи, порядок, рівень деталізації.
- **Правила комунікації** — чого не писати клієнтам, як звертатися, які дані не розкривати.

Оформлення власних скілів описано в офіційній документації: [docs.claude.com](https://docs.claude.com/en/resources/prompt-library).
